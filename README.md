Supplementary Data Package for GDEval-LLM

This supplementary package accompanies the manuscript:

GDEval-LLM: An Evidence-Grounded Multi-Stage Framework for Assessment Support and Pedagogical Feedback in Graphic Design Education

Included files

1. Supplementary_Materials.docx
   This document reports model settings, prompt templates, scoring and feedback-generation procedures, requested JSON schemas, stored output fields, and availability notes for generated outputs.

2. run_eval.py
   This file contains the implementation used to run the model-based scoring and standardized feedback-generation procedures. It includes method-specific scoring prompts, the shared final Korean feedback-generation prompt, retry handling, image preprocessing, and output parsing logic.

3. rubric.json
   This file contains the rubric definitions and anchor descriptors used at runtime for the five criteria:
   C1 Visual Hierarchy & Layout
   C2 Typography
   C3 Color
   C4 Concept & Communication
   C5 Craft & Finish

4. assignment_brief.txt
   If included, this file contains the assignment brief supplied to expert raters, LMM prompts, and feedback evaluators. In the reported Claude Opus runs, the same assignment brief text was inserted into all method-specific scoring prompts and the standardized final feedback-generation prompt.

5. expert_ratings_anonymized.csv
   This file contains the raw anonymized expert ratings used for the expert-agreement benchmark and the model–expert score-agreement analyses. Each row corresponds to one work, one expert rater, and one rubric criterion.

   Expected columns:
   work_id
   expert_id
   criterion
   score

   Expected structure:
   50 works × 3 expert raters × 5 criteria = 750 rows

6. feedback_ratings_anonymized.csv
   This file contains the anonymized blind feedback-evaluation ratings used for the feedback-quality analyses. Each row corresponds to one work, one feedback evaluator, one assessment method, and one feedback-quality dimension.

   Expected columns:
   work_id
   rater_id
   method
   dimension
   rating

   Expected structure:
   50 works × 3 methods × 4 dimensions × 3 raters = 1800 rows

7. feedback_outputs_claude_with_lengths.csv
   This file contains the final standardized Korean feedback outputs generated by Claude Opus for all 50 works and all three assessment methods. Each row corresponds to one work-method pair and includes the final student-facing feedback text, the initial method-specific feedback text, the information supplied to the standardized final feedback-generation prompt, and the character length of the final feedback.

   Expected columns:
   work_id
   method
   repeat
   overall_score
   feedback
   feedback_length_chars
   initial_feedback
   feedback_generation_input
   provider
   model
   temperature
   assignment_brief_provided

   Expected structure:
   50 works × 3 methods = 150 rows

8. model_scores_claude.csv
   This file contains the Claude Opus outputs for all 50 works and all three assessment methods. It includes model-generated scores, final standardized feedback text, initial method-specific feedback text, feedback-generation input, criterion-level evidence and justifications where available, and configuration fields.

   Expected structure:
   50 works × 3 methods = 150 rows

9. model_scores_gpt4o.csv
   This file contains GPT-4o score outputs used only for the score-agreement replication analysis. GPT-4o was not used in the blind feedback-quality evaluation, and no GPT-4o feedback-output file is required for reproducing the reported feedback-quality results.

   Expected structure:
   50 works × 3 methods = 150 rows

   Note. This file was created by combining the GPT-4o outputs from results_direct.csv, results_rubric.csv, and results_gdeval.csv. These outputs are used only for model–expert score-agreement replication, not for the blind feedback-quality evaluation.

10. analysis_script.py
   If included, this file contains the statistical analysis code used to compute inter-expert agreement, model–expert score agreement, feedback-quality statistics, post-hoc comparisons, and robustness checks.

Anonymization and privacy

Student design images are not included in this supplementary package because they are subject to educational privacy and consent restrictions. The uploaded data files use stable anonymized work identifiers such as image01–image50. Expert raters and feedback evaluators are anonymized using generic identifiers.

Notes on feedback evaluator IDs

The feedback evaluator identifiers in feedback_ratings_anonymized.csv were anonymized as Evaluator_1, Evaluator_2, and Evaluator_3. These identifiers do not correspond to evaluator names or original file names.

Notes on model use

Claude Opus was used for the primary experiment, including score generation and the final standardized feedback outputs used in the blind feedback-quality evaluation. GPT-4o was used only to replicate the score-agreement analysis. Therefore, model_scores_gpt4o.csv is included for score-replication transparency, but GPT-4o feedback outputs are not included and are not required for reproducing the reported feedback-quality results.

Notes on generated feedback outputs

feedback_outputs_claude_with_lengths.csv contains the final standardized Korean feedback outputs used for the blind feedback-quality evaluation. The methods differ in the assessment information supplied to the shared final feedback-generation prompt:
- Direct scoring: holistic score and initial direct feedback text
- Rubric-based prompting: five criterion scores and initial rubric-based feedback text
- GDEval-LLM: visual observations, five criterion scores, criterion-specific visual evidence, and evidence-based justifications

Data availability

Student design images are not publicly available due to privacy and consent restrictions. De-identified expert ratings, anonymized feedback-evaluation ratings, Claude Opus score and feedback outputs, GPT-4o score-replication outputs, final generated feedback texts, prompt templates, rubric definitions, and implementation code are provided in this package. Additional raw provider responses, execution logs, and analysis scripts may be made available from the corresponding author upon reasonable request, subject to institutional and consent restrictions.

Recommended citation in manuscript

The supplementary package includes prompt templates, rubric definitions, implementation code, anonymized expert ratings, anonymized feedback-evaluation ratings, Claude Opus score outputs, GPT-4o score-replication outputs, and final generated feedback outputs. Student design images are not shared because of privacy and consent restrictions.
